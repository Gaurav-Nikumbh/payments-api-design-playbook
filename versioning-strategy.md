# API Versioning Strategy

Payment APIs must evolve without breaking existing integrations.

## Common Approaches

- URL versioning (/v1/payments)
- header-based versioning

---

## Recommended Approach

Contract-first versioning with backward compatibility.

---

## Key Practices

- avoid breaking changes
- use clear deprecation timelines
- communicate changes early

---

## Impact

Good versioning improves integration success and reduces support overhead.
